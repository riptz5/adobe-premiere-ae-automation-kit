# Security Policy

## Reporting a Vulnerability

Por favor **no** abras un issue público para reportar vulnerabilidades.

Usa GitHub Security Advisories:
- Repo → **Security** → **Report a vulnerability**

Incluye:
- Descripción del problema y alcance
- Pasos para reproducir
- Impacto estimado
- Versiones/commit afectado (si aplica)

## Scope

Este repo incluye scripts, paneles y un servidor local. En general, asume que cualquier manejo de tokens (Frame.io/LLM) debe vivir en `.env` y nunca en commits.
